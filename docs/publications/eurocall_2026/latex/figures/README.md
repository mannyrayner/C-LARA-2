# Figure descriptions

The submitted EuroCALL paper's figures are represented in this repository by
verbal descriptions so the text-only pull-request workflow remains usable.
The LaTeX source's original `\includegraphics` references are preserved as
historical source references; these descriptions are the report-writing
context available in this repository.

1. **Picture-dictionary editor.** A specialised dictionary-entry interface for
   French entries such as *aimer*, *attraper*, and *avoir*. Each row shows the
   word, lemma, part of speech, gloss/translation, optional human suggestion,
   image-generation prompt, and selected illustration, with bulk selection and
   generation controls.
2. **Picture-clue word scramble.** A learner-facing French exercise showing a
   cartoon snail. The learner selects letters in a grid to form *ESCARGOT*;
   controls include clearing/showing the answer, checking the selection, and
   moving to the next item.
3. **Mandarin/pinyin Assistant question.** A project-understanding query asks
   whether Mandarin texts can be produced and whether pinyin annotations can
   be added.
4. **Mandarin/pinyin Assistant answer.** The answer confirms support and gives
   repository links for language choices, text generation, tokenisation,
   romanisation/pinyin processing, manual editing, HTML ruby rendering, and
   the `pypinyin` runtime dependency.
5. **Haitian Creole audio Assistant question.** A query asks whether a user can
   record and use their own audio because the platform's TTS sounds French and
   is unsuitable for Haitian Creole.
6. **Haitian Creole audio Assistant answer.** The answer distinguishes current
   TTS/no-audio support from the planned community recording/upload workflow,
   identifies the relevant roadmap and `ISSUE-0026`, and recommends the
   currently available no-audio/skip-TTS workaround.
