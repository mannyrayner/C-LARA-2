# 1. Introduction and History (Outline)

## 1.1 Purpose of this report

- Provide a first consolidated progress report on C-LARA-2.
- Summarize platform capabilities, development methodology, current status, and near-term priorities.
- Create a shared basis for refining later publication outputs.

## 1.2 Project lineage: LARA → C-LARA → C-LARA-2

- **LARA** established the core pedagogical idea: richly annotated, multimodal language-learning texts.
- **C-LARA** integrated stronger AI assistance for text and annotation workflows, and introduced broader end-user content generation capabilities.
- **C-LARA-2** is a reimplementation with architecture rationalization, expanded feature scope, and a more explicit AI-centered engineering workflow.

## 1.3 Why reimplementation was necessary

- Legacy formats and implementation patterns limited maintainability and extension velocity.
- Broader language support and robust human-in-the-loop editing required cleaner internal abstractions.
- Growth in platform scope (images, exercises, communities, import/migration, deployment) required a more systematic software process.

## 1.4 Current reporting frame

- C-LARA-2 should be treated as both:
  - a language-learning content platform, and
  - an ongoing experiment in AI-authored, repository-native software engineering.
