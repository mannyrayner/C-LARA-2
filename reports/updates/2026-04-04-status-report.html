C-LARA-2 status report (2026-04-04)

Summary for non-experts

C-LARA-2 now supports an end-to-end workflow for creating language projects: authoring text, running annotation/compilation, adding image assets, generating exercises, and publishing content in the Django platform.

What users can do now

  • Create and manage projects with multiple processing options.
  • Run compile jobs with monitor/status pages and task updates.
  • Manage project image styles, image elements, and page images.
  • Generate and publish cloze/flashcard exercise sets.
  • Browse published content and view associated exercises.
  • Use improved MWE prompting resources across multiple languages.

Current technical strengths

  • Richer AI client support (text/json/image helpers, telemetry, retries, heartbeat patterns).
  • Generalized romanization options and resilient segmentation/tokenization fallbacks.
  • Improved MWE consistency across prompt templates and postprocessing.
  • More robust audio generation/caching and compile-time asset handling.

Near-term focus areas

  • Keep improving report quality and communication for non-technical stakeholders.
  • Continue tightening language-specific prompt guidance and evaluation loops.
  • Stabilize and simplify operational tooling around compile monitoring and admin workflows.
